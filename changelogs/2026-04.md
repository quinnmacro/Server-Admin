# 变更日志 - 2026年4月

## 2026-04-12

### SSH 性能优化系统

#### 新增
- **SSH 性能优化框架**：
  - `ssh-benchmark.sh`：SSH 性能基准测试脚本，支持响应时间、传输速度、并发连接测试
  - `ssh-optimize.sh`：SSH 配置管理脚本，支持部署、验证、回滚、状态检查
  - `deploy-ssh-config-test.sh`：部署前测试脚本
  - `ssh-performance.conf`：SSH 服务器性能配置模板

- **Telegram 机器人 v3.0**：
  - 重新设计 3×3 主菜单布局
  - 新增 "🎉 趣味" 菜单，包含 8 个趣味功能（笑话、彩蛋、游戏等）
  - 新增 SSH 性能监控命令：`/sshstatus`, `/sshperformance`, `/sshoptimize`, `/sshdiagnose`, `/sshhistory`, `/sshconfig`
  - 改进的菜单导航和回调系统

#### 优化
- **SSH 客户端配置全面优化**：
  - 启用连接复用（ControlMaster auto），改进 91%
  - 配置现代加密算法（chacha20-poly1305, aes128-gcm, curve25519 优先）
  - 性能参数调优（Compression delayed, TCPKeepAlive yes, IPQoS lowdelay throughput）
  - 测试结果：后续连接速度提升 11 倍

- **系统监控增强**：
  - 扩展健康检查系统，集成 SSH 性能监控
  - 改进的 Telegram 通知格式和内容
  - 统一的配置管理

#### 修复
- **Telegram 机器人菜单显示问题**：修复返回主菜单时不显示完整欢迎消息的问题
- **备份脚本路径问题**：修正备份目录路径和文件扩展名

## 2026-04-11 (下午)

### 优化

#### 系统性能调优
- 创建 `/etc/sysctl.d/99-server-optimize.conf` 内核优化配置
- 降低 Swap 使用倾向（swappiness: 60 → 10）
- 优化 TCP 连接效率（Fast Open、连接复用、缓冲区）
- 提高系统稳定性和网络性能

#### Docker 容器资源限制
- SanctionList backend: 内存限制 256MB，CPU 0.5核
- SanctionList frontend: 内存限制 64MB，CPU 0.25核
- Homepage: 内存限制 64MB，CPU 0.25核
- 防止单个容器耗尽系统资源

#### 日志管理
- Docker 容器日志轮转配置（最大 10MB，保留 7 天）
- systemd-journald 日志限制（最大 100MB，保留 7 天）

#### 监控增强
- 健康检查脚本 v2.1
- 新增 OOM（内存溢出）检测
- 新增 TCP 连接状态统计
- 新增容器内存使用率监控
