# ADR-001: 监控系统选择

## 状态

已接受

## 背景

服务器只有 2GB 内存，需要选择合适的监控方案。

## 决策

选择轻量级 Shell 脚本 + Cron 方案，而非 Prometheus + Grafana。

## 理由

1. **资源消耗**: Prometheus + Grafana 需要约 300MB 内存，占服务器 15%
2. **简单性**: Shell 脚本无额外依赖，易于维护
3. **足够性**: 当前监控需求简单，不需要复杂可视化
4. **通知**: Telegram 通知足够满足告警需求

## 替代方案

| 方案 | 内存占用 | 优点 | 缺点 |
|------|----------|------|------|
| Shell + Cron | ~5MB | 轻量、无依赖 | 无可视化 |
| Prometheus + Grafana | ~300MB | 可视化、强大 | 资源消耗大 |
| Uptime Kuma | ~50MB | Web UI | 需要额外部署 |

## 后果

- 监控功能简单，无历史趋势图
- 维护成本低
- 适合小型服务器
